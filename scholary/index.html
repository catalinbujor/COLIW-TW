Abstract

COLIW (Command Line for Web) este o aplicatie web care isi propune sa ii ofere utilizatorului interactiune cu servicii sociale web la linia de comanda.

Utilizatorii pot astfel sa se logheze pe diferite platforme si sa execute diferite comenzi. Folosindu-se de o implementare modulara, COLIW le pune utilizatorilor la dispozitie o platforma extensibila capabila sa ofere suport pentru realizarea de mash-up-uri exclusiv cu ajutorul unui set de mini-comenzi, in stilul celor oferite de bash.

Architecture

Microservices reprezintă o metoda distinctiva de concepere a sistemelor software. Fiecare functionalitate sau serviciu este independentă, poate fi implementată, mica și modulara, ceea ce înseamna ca fiecare dintre ele este la fel de decuplat posibil de la alta. Fiecare microservice din aplicația noastra ruleaza pe un server separat de server-ul central iar unificarea acestora este realizata de componenta router prin intermediul cailor.

  • Twitter microservice -acest microserviciul este menit sa ofere functionalitatile oferite de twitter api.

  • dbModel microservice -acest microserviciul este menit sa expuna baza de date.

Client-side

The User Interface aplicatiei este realizata folosind HTML5 si CSS3 vanilla.

Design Details

  • Fiind o apicatie web orientata pe micro servicii, fiecare functionalitate exista intr-un server separat.Astfel, o interogare va apela la o entitate diferita, dupa capabilitatile acesteia.Dupa ce aceasta este returnata cu succes, sesiunea userului se updateaza(daca userul este logat si exista sesiune).

  • Main View La fiecare comanda tastata, se creeaza un nou obiect de tipul terminal.

  • The user must register and generate a JWT before using the REST API exposed by MuSeR. Every subsequent call of the API will require the token.

  • The 'preferred' data transfer MIME type of MuSeR is application/json for the ease of use and the nice integration with Javascript and NodeJS.

  • Each instance's view for an entity type (e.g. an Artist or a Song) will smartly suggest other instances of the same entity type taking into consideration different aspects related to it: genre, time period, preferences etc.

  • Playlists can be accessed via a SPARQL endpoint that will provide a way of accessing the data from the RDF Storage (GraphDB). They can also be imported into / exported from MuSeR under the JSPF format.
